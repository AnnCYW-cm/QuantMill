    QuantMill Roadmap¶

    v1.0.0 已封为量化研究验证平台。下一阶段目标不是堆更多模型,而是把研究链路继续推进到更可靠的数据治理、实验复现、前瞻验证和交易执行基础设施。

    V1.0 当前边界¶

    已完成:

    • 可插拔数据层:bars / fundamentals / universe / quotes 四接口、缓存、回退链、自备数据模板。
    • 可插拔模型层:分类/回归 ModelProvider、LightGBM 默认实现、logistic/ridge 替代实现。
    • 横截面研究主线:面板、IC、稳健因子组合、ML 排名对照、top-k 回测。
    • 可信度层:DSR/PBO、跨市场验证、前视/幸存者偏差量化、严格 PIT 测试。
    • 前瞻纸面记录:只前进、不回看,用于沉淀真实 live track record。
    • Web/CLI/实验 YAML:能从命令、网页和配置文件运行主流程。

    暂不承诺:

    • 全自动实盘资金托管。
    • 多券商生产级订单路由。
    • 完整账户对账和异常恢复。
    • 机构级商业风险模型。
    • 免费数据无法提供的完整 PIT 成分史和历史新闻覆盖。

    V1.1 数据治理¶

    目标:让数据源替换不仅"能接",而且"可审计、可复现、可比较"。

    • 数据集元数据:记录 provider、版本、下载时间、复权口径、时区、币种、交易日历。
    • 缓存 manifest:每个缓存文件带 schema hash、时间范围、源名称和刷新策略。
    • 数据质量检查:缺口、重复日期、异常价格、复权跳变、停牌、涨跌停、成交量异常。
    • PIT universe 完整化:支持带 in_date/out_date 的真实成分史导入,修掉"后被踢出"的幸存者偏差。
    • 数据源对比报告:同一标的在 akshare/yfinance/自备源之间的价格差异和缺失差异。

    交付标准:

    • quantmill data audit 输出数据质量报告。
    • 所有面板缓存可追溯到具体 provider 和生成参数。
    • 自备付费数据接入后不需要改 cross/model/portfolio 调用点。

    V1.2 实验注册与模型治理¶

    目标:每一次回测和模型结果都能复现,并能回答"这条曲线从哪里来的"。

    • Experiment Registry:记录 config、代码 commit、数据 manifest、因子列、模型 provider、参数、随机种子。
    • Model Registry:保存训练产物、模型签名、feature schema、训练窗口、验证指标。
    • 因子版本:把因子表达式、窗口参数和中性化方式记录成可比较版本。
    • 结果对比页:按实验维度比较年化、超额、DSR、PBO、换手、回撤、跨市场表现。
    • 防 p-hacking 记录:显式记录同一研究主题下尝试次数,让 DSR 的 n_trials 有真实来源。

    交付标准:

    • quantmill experiment run 生成可复现实验包。
    • 任意历史实验能用原 config 重新运行并得到同级别结果。
    • Web 中有实验列表和对比视图。

    V1.3 前瞻验证产品化¶

    目标:把"回测会骗人"落到持续运行的 live track record。

    • 多策略前瞻账户:同一市场跟踪 composite、ml、不同 k、不同风控参数。
    • 回测 vs 前瞻并排:同一策略的历史回测曲线和真实前瞻曲线分开展示。
    • 前瞻完整性检查:缺失日期、价格命中率、换仓日、成本、回撤开关触发记录。
    • 每周/月报:自动汇总前瞻收益、最大回撤、换手、持仓变化、跑赢基准比例。
    • 策略晋级规则:只有前瞻曲线积累足够样本后,才允许进入更高风险层。

    交付标准:

    • quantmill forward status 能输出可审计的前瞻运行摘要。
    • Web 前瞻页展示每条 live track record 的起始日、净值点数、当前持仓和完整性状态。
    • 前瞻历史点保持 append-only 测试锁死。

    V1.4 执行与风控底座¶

    目标:从纸面执行推进到可控的 broker adapter 框架,但仍默认纸面盘。

    • 订单状态机:submitted / accepted / partially_filled / filled / canceled / rejected。
    • 前置风控:单票上限、行业/市场暴露、日内亏损限额、最大下单金额、禁止交易清单。
    • 账户对账:本地目标持仓 vs broker 持仓 vs 现金/净值。
    • 下单模拟器增强:部分成交、滑点、拒单、交易日历、涨跌停/停牌处理。
    • Alpaca adapter 完整化:纸面盘下单、撤单、成交回报、错误处理。
    • Kill switch:异常波动、数据缺失、价格过期、账户不一致时停止下单。

    交付标准:

    • quantmill paper run 和 broker adapter 共用同一个订单/风控契约。
    • 任意实盘适配器必须先通过离线 broker contract tests。
    • 默认仍不自动实盘;实盘入口需要显式 broker 和密钥配置。

    V2 候选方向¶

    • 多资产扩展:ETF、期货、数字资产,但先统一交易日历和成本模型。
    • 更完整的基本面:ROE、成长、质量、盈利修正、财报披露延迟。
    • 新闻/公告历史库:让 LLM 文本因子真正进入 PIT 回测。
    • 机构级风险模型接口:接商业风险模型或自备行业/风格暴露。
    • 插件生态:第三方 DataProvider、ModelProvider、BrokerProvider 的标准模板和测试套件。

    不做清单¶

    • 不把单次漂亮回测包装成可赚钱结论。
    • 不默认开启真实下单。
    • 不用当前成分股假装无幸存者偏差。
    • 不让 LLM 直接预测涨跌;LLM 只做文本结构化和分类。
    • 不为追求短期指标牺牲 PIT、walk-forward 和前瞻记录纪律。
