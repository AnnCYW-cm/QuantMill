    产品重新设计 · AI 量化交易平台¶

    定位:开源、全链条、多市场的 AI 量化研究与交易平台;架构对齐微软 Qlib,以"多市场(港/美/A)一等公民 + 可信度/抗过拟合内建 + 上手友好双语"为立身之本。 本文是设计蓝图(北极星架构 + 个人可走的分阶段路径),不是一次性实现清单。

    0. 一句话定位¶

    "一个诚实的、全链条的开源 AI 量化平台:从多市场数据到组合执行,每一步都内建防过拟合的可信度检验——让你先知道策略到底成不成立,再谈上真钱。"

    对标 Qlib 的完整度,但补上 Qlib 的三个空当。

    1. 为什么这个项目该存在(和 Qlib 的关系)¶

    微软 Qlib 本平台的差异化空当
    完整度 数据→模型库→风险→组合→RL执行,链条最全 对齐它的链条(北极星),不重造轮子
    市场 region 仅 cn/us 原生,港股要自己接,A股制度细节要自己处理 港/美/A 股一等公民,双数据源容错,A股涨跌停/T+1 内建
    抗过拟合 有 walk-forward,但不把"这策略是不是运气"做进主流程 可信度层内建:DSR/PBO/广度/稳健性成为工作流必经一站
    上手 学习曲线陡、偏研究、英文 开箱即用 CLI+仪表盘、中英双语、渐进式
    维护 微软,节奏偏慢/研究导向 社区驱动、插件化、贴合个人实盘

    结论:不做"更差的 Qlib",做"多市场 + 诚实 + 好上手的 Qlib 同级平台"。

    ✅ 命名已定:quantmill(2026-07-04 敲定)。quant + mill(加工厂/磨坊)= "把原始数据加工成策略的量化工厂",呼应全链条平台定位;PyPI 空、无同域活跃撞名。原名 quantlib 撞著名 C++ 库 QuantLib,已弃用。

    2. 目标用户与场景¶

    • 主用户:会 Python、想做多市场(尤其 A/港股)量化研究的个人/学习者/小团队;能 fork、能扩展。
    • 典型场景:
    • 研究:抓数据 → 造因子 → 训模型 → 组合回测 → 可信度体检 → 报告。
    • 迭代:换标的/因子/模型/参数,批量验证是否稳健(而非单只运气)。
    • 监控:自选股每日信号面板。
    • (进阶)接模拟盘/实盘执行闭环。
    • 非目标:高频/做市、机构级另类数据、纯无代码小白(那是 BigQuant/果仁的地盘)。

    3. 设计原则(写进项目宪法)¶

    1. 无未来函数是地基,且用测试焊死(已做,继续扩)。特征只用过去、只有 label 用未来。
    2. 诚实优先:任何回测结果默认配一份可信度体检(DSR/PBO/广度);报告永远显示 vs 买入持有 + 试验次数。
    3. 插件化 + 配置驱动:数据源、因子、模型、组合器、执行器都是可替换插件;一条 YAML/CLI 描述完整工作流(学 Qlib 的 workflow)。
    4. 多市场一等公民:统一接口下,港/美/A 股(+加密)都能跑;市场制度差异(涨跌停、T+1、交易日历)在框架层处理。
    5. 复用而非重造:回测底层可包 backtesting.py/vectorbt;深度模型、优化器用成熟库(PyTorch、cvxpy)。重资产依赖设为可选。
    6. 渐进式:小白一条命令能跑通;高手能替换每一层。中英双语贯穿。

    4. 北极星架构(全链条 13 层)¶

    ┌─ 编排层 Orchestration ──────── 配置驱动工作流 + CLI + 实验记录 ─┐
│                                                                │
│  ① 数据 Data ──────────── 多市场OHLCV/基本面/另类, PIT, 缓存, 复权, 日历/股票池
│  ② 因子 Factor ────────── 因子表达式引擎 + 因子库(→Alpha158级) + IC/IR分析
│  ③ 标注 Label ─────────── 可配置(收益方向/分位/中性化), 无未来函数
│  ④ 模型库 Model Zoo ───── GBDT/线性/深度(LSTM/Transformer/GNN,可选依赖), 统一接口
│  ⑤ 预测 Signal ────────── 截面打分/排序 + 时序CV/walk-forward
│  ⑥ 风险模型 Risk ──────── 波动率/相关性/暴露约束(Barra-lite)
│  ⑦ 组合 Portfolio ─────── 信号→权重: TopK/等权/逆波动/均值方差优化(cvxpy)
│  ⑧ 回测 Backtest ──────── 组合级, 含成本滑点, A股制度(涨跌停/T+1)
│  ⑨ 执行 Execution ─────── 模拟盘/实盘: Alpaca(美) / QMT·PTrade(A/港), 订单管理
│  ⑩ 可信度 Credibility ★── DSR/PBO/广度/稳健性/多重检验校正 —— 我们的护城河
│  ⑪ LLM/另类 ★(前沿) ──── 新闻/财报→特征, LLM因子助手(RD-Agent式)
│  ⑫ 呈现 Presentation ──── 单只报告/信号面板/工作台首页/实验对比
│  ⑬ 评估 Evaluation ────── 指标(夏普/回撤/IC/vs基准) + 归因
└────────────────────────────────────────────────────────────────┘

    5. 现状映射:quantmill 已有 vs 要补¶

    层 目标能力 现状 动作
    ① 数据 多市场+PIT+复权+日历+基本面 ✅ 港/美/A OHLCV+缓存+双源回退 🟡 加基本面、交易日历、PIT 严格化
    ② 因子 表达式引擎+因子库+IC分析 🟡 18 个硬编码指标 🔴 加因子表达式 DSL + 因子库 + IC/分位分析
    ③ 标注 可配置多目标 🟡 未来N日涨跌 🟡 加分位/中性化/回归目标
    ④ 模型库 可插拔多模型 🟡 仅 LightGBM 🟠 抽象统一接口,GBDT先全,深度设可选
    ⑤ 预测 截面排序+CV ✅ 时序CV+walk-forward 🟢 达标,补截面排序
    ⑥ 风险模型 波动/暴露约束 ❌ 仅最大回撤 🟠 Barra-lite
    ⑦ 组合 信号→权重优化 ❌ 单标的 🔴 关键缺口:组合构建层
    ⑧ 回测 组合级+A股制度 🟡 单标的+成本 🟠 升级为组合级,加涨跌停/T+1
    ⑨ 执行 模拟/实盘 ❌ 🟠 后期:Alpaca / QMT 适配器
    ⑩ 可信度 ★ DSR/PBO/广度 ✅ 广度+稳健性(已领先) 🔴 补 DSR/PBO,升级成统计严谨
    ⑪ LLM/另类 ★ 消息面特征 ❌ ⚪ 前沿,谨慎(见调研:LLM无稳定alpha)
    ⑫ 呈现 报告/面板/首页 ✅ report/dashboard/home 🟢 达标
    ⑬ 评估 指标+归因 🟡 vs买入持有 🟡 加 IC/归因
    编排 配置驱动工作流 🟡 CLI+config 🟠 加 YAML 工作流描述

    一句话:数据/预测/呈现/可信度已达标或领先;三个红色关键缺口 = 因子引擎、组合构建、DSR。

    6. 分层设计要点(公共接口草案)¶

    ① 数据层 data/:get_ohlcv(已有)升级为 DataHandler,统一 OHLCV + 基本面 + 日历;新增 Calendar(交易日历/停牌)、Universe(动态股票池)。插件式数据源 Source(yfinance/akshare/自定义)。

    ② 因子层 factor/(新): - expr:因子表达式引擎,支持 Rank(close/Mean(close,20)-1)、Corr(vol,close,10) 这类 DSL(仿 WorldQuant/Qlib Alpha158)。 - library:内建因子集(把现 18 个扩到 ~100+,分动量/价值/波动/量能/情绪)。 - analysis:IC/RankIC、分位组合、因子衰减(alphalens 式)。

    ③ 模型层 model/:抽象 Model.fit/predict;GBDTModel(LightGBM/XGB/Cat)、LinearModel;torch 后端的 LSTM/Transformer/GAT 设为 extras(可选依赖)。cv:时序CV/walk-forward(已有,下沉为公共组件)。

    ⑥⑦ 风险+组合层 portfolio/(新,最关键): - RiskModel:协方差估计、波动率目标、暴露约束。 - Optimizer:TopK、EqualWeight、InverseVol、MeanVariance(cvxpy),统一 signals → weights 接口,含换手/集中度/仓位上限约束。

    ⑧ 回测层 backtest/:从单标的升级到组合级;Market 抽象处理 A股涨跌停/T+1、港股/美股规则;底层可继续包 backtesting.py 或引入向量化引擎。

    ⑩ 可信度层 credibility/(护城河,升级现 validate.py): - dsr:去膨胀夏普(校正多重检验选择偏差 + 非正态)。 - pbo:回测过拟合概率(CSCV 组合对称交叉验证)。 - breadth/robustness:现有,保留。 - trials:自动记录"试了多少参数组合 N"并纳入显著性判断。 - 设计成"回测跑完自动出体检报告",而非可选步骤。

    ⑨ 执行层 execution/(后期):Broker 适配器 AlpacaBroker(美)、QMTBroker(A/港);先纸面交易闭环。

    ⑪ LLM 层 llm/(前沿,可选):新闻/财报 → 情绪/事件特征;LLM 因子助手。强制走可信度层验证——调研已证 LLM 智能体无稳定 alpha,不做过度承诺。

    编排 workflow/:YAML/dict 描述一条完整流水线(数据→因子→模型→组合→回测→体检→报告),quant run config.yaml 一键复现。

    7. 分阶段路线图(个人可走,每阶段都有可用产物)¶

    • Phase 0 · 地基重构(1–2 天):改名去 QuantLib 冲突;把现有模块重排成分层架构(data/factor/model/portfolio/credibility/...);定义各层插件接口;测试全绿。
    • Phase 1 · 诚实内核(先做,性价比最高):① 可信度层加 DSR + PBO + 试验次数记录;② 因子表达式引擎 + 因子库扩到 ~50–100;③ 因子 IC 分析。→ 立刻拉开与 To C 产品的差距。
    • Phase 2 · 组合化(从"选股玩具"变"策略"):风险模型 + 组合优化器 + 组合级回测 + A股制度。
    • Phase 3 · 模型库与工作流:模型抽象 + GBDT 全家桶,深度模型设可选;YAML 配置驱动工作流。
    • Phase 4 · 执行闭环:纸面交易(Alpaca / QMT),实时数据→信号→模拟下单。
    • Phase 5 · 前沿:LLM/消息面特征(强制过可信度层)。
    • Phase 6 · 开源化:文档站、可复现 benchmark、示例库、CONTRIBUTING、CI、语义化版本、清晰公共 API。

    现实提醒:Phase 0–2 是个人 1–2 周能出彩的部分,也最能立住"诚实的组合级平台"人设;Phase 3+ 是长期。别一上来追 Qlib 全家桶,先把"因子→组合→可信度"这条最有辨识度的短链打磨到亮。

    8. 项目结构与命名¶

    命名已定:quantmill(quant + mill 加工厂,呼应全链条平台;PyPI 空、无同域撞名)。

    9. 关键取舍与风险(诚实清单)¶

    1. 别真去对标 Qlib 全家桶:微软有团队、有算力。我们赢在"多市场 + 诚实 + 好用",不在"模型最多"。深度模型/RL执行设为可选或后期,避免摊薄。
    2. 组合优化是从玩具到平台的分水岭,也是工程量真正上台阶的地方(cvxpy/换手约束/数值稳定)。
    3. A股制度细节是脏活但也是差异化:涨跌停、停牌、T+1、复权,做对了就是护城河,做错了回测全是假的。
    4. LLM/另类数据谨慎:调研已证其无稳定 alpha,做成"可选特征源 + 强制可信度验证",而非卖点承诺。
    5. 开源要活靠社区:文档、示例、可复现 benchmark、清晰 API 比功能数量更重要;先把一条"因子→组合→体检"的链讲成教科书级示例。
    6. 范围现实主义:这是多阶段长期设计。北极星是全链条,但每个 Phase 都要能独立交付、独立可用。

    11. 横截面选股 + 诚实结论(2026-07 主线)¶

    11.1 为什么升级到横截面¶

    原本"每只票各训一个模型预测自己涨跌"学不到"今天买 A 还是 B"。对标 Qlib/Numerai,真正的选股是横截面:全市场一个模型每天给所有票打分排名。cross/ 模块补上了这个范式缺口(详见 docs/ARCHITECTURE.md §5 数据流)。

    11.2 两种策略,一条铁律¶

    • 稳健因子组合(主推):ey+bp+dist_high_20+pos_60d−vol_20d 固定配方、零训练。跨 A股/港股都为正、DSR 0.989。这是产品里可信度最高的策略。
    • ML 排名(对照反面教材):43 因子 LightGBM。A股样本内漂亮(+33.5%),但港股翻车(−10.2%)、去前视偏差后崩成负(−7.7%)。
    • 铁律:能跨市场活下来的简单东西,胜过看着漂亮的黑箱 ML。

    11.3 可信度层扩容¶

    可信度不再只是 DSR/PBO,新增两道跨市场/偏差关卡,成了四道照妖镜: 1. DSR/PBO(样本内多重检验) 2. 子区间衰减(regime 依赖) 3. 跨市场验证 cross validate(A股↔港股同号/都为正) 4. 前视/幸存者偏差量化 cross survivorship(全池 vs PIT池)

    完整调查结论见 docs/RESEARCH_NOTES.md:复杂 ML 是过拟合幻觉,简单价值/低波/动量因子是"真但弱"的边际,不可重仓。 这份"能自己证伪自己"的诚实,是 quantmill 在满屏营销的量化圈里真正的立身之本。
